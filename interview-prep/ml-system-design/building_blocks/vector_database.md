# Vector Database

Stores high-dimensional vectors (embeddings) and supports efficient **approximate nearest neighbor (ANN)** search — finding the most similar items to a query.

---

## Why a Dedicated DB?

Regular DBs support exact lookups (find row where id = 42). Finding "the 10 most similar rows" requires computing distance to every row — O(N) per query, which is too slow at millions of records.

Vector DBs use specialized indexing (HNSW, IVF) to find approximate nearest neighbors in O(log N).

---

## What Are Embeddings?

Dense numerical vectors that encode semantic meaning:
- Text → sentence embedding: "good movie" and "great film" map to nearby vectors
- Images → visual embedding: similar-looking images cluster together
- Users, products → entity embeddings for recommendations

Generated by an embedding model (BERT, CLIP, OpenAI Embeddings API, etc.).

---

## Core Operations

| Operation | Description |
|---|---|
| **Upsert** | Store a vector with an ID and optional metadata |
| **Query (ANN search)** | Given a query vector, return top-K nearest neighbors |
| **Filter** | ANN search + metadata filter (e.g., "similar items where category = 'shoes'") |
| **Delete** | Remove a vector by ID |

---

## Indexing Algorithms

### HNSW (Hierarchical Navigable Small World)
- Graph-based. Builds a multi-layer graph of vectors.
- Very fast queries, good recall.
- High memory usage.
- Used by: Weaviate, Qdrant, pgvector.

### IVF (Inverted File Index)
- Clusters vectors; at query time, search only nearby clusters.
- Lower memory than HNSW.
- Requires a training step to build clusters.
- Used by: Faiss, Pinecone.

**Recall vs speed trade-off**: ANN is approximate — tuning the index trades recall (finding the true nearest neighbor) against query latency.

## Common Use Cases

- **Semantic search**: find documents semantically similar to a query
- **Recommendation**: "users like you also liked…" based on user embeddings
- **RAG (Retrieval-Augmented Generation)**: retrieve relevant context for an LLM
- **Duplicate/near-duplicate detection**: find similar images, products, or documents
- **Anomaly detection**: find items far from all known clusters

---

## Trade-offs

| Pro | Con |
|---|---|
| Fast similarity search at scale | Approximate — not exact results |
| Handles high-dimensional data | Embeddings must be pre-computed and kept fresh |
| Supports hybrid search (vector + metadata filter) | Index rebuild needed when data changes significantly |
| Decoupled from ML model | Another system to maintain |
